# /docs/product_description/04_who_it_is_for.md

## Primary Users

### Solo developer or student builder

This user wants fast help understanding an unfamiliar code path and getting to a first plausible implementation.

They care about:
- low setup friction
- low cost
- understandable outputs
- support for common codebases

### Developer handling a small feature or fix request

This user wants a fast answer to "what should change?" without reading the entire repository first.

They care about:
- relevant file selection
- narrow plans
- reviewable diffs
- clear rationale

## Secondary Users

### Demo or teaching audience

This user values legibility. Intermediate outputs matter because they show how the system arrived at an answer.

### Future maintainers of RepoMind

This user needs the product idea to stay coherent even as implementation details change.

## Non-Target Users

RepoMind is not primarily designed for:
- enterprise-scale repository platforms
- massive monorepos
- full autonomous engineering workflows
- security-critical auto-patching without human review
