## Backlog Features

1. `.ipynb` support for notebook ingestion and code-cell chunking
2. Fast LLM agentic pipeline
3. Logging for debugging
4. Repository-level caching for repeated runs
5. Dedicated CLI entrypoint for pipeline execution
6. Retrieval confidence signal
7. Better structured error states beyond stage-based failures
8. Expanded unit, integration, and e2e coverage
9. User selection of retrieved files before generation
10. Safe review flow for applying proposed patches
11. Repository summaries for faster first-run orientation
12. Broader language coverage
13. Comparison of multiple generation models for the same plan
14. Lightweight evaluation datasets for regression testing
