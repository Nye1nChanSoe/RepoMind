# /docs/iterations/00_README.md

## Purpose

This folder records implementation iterations for RepoMind improvements.

Each iteration should document:
- the goal of the iteration
- what changed
- what was validated
- what remains for the next pass

## Usage Rule

- treat iteration docs as change history, not as stronger guidance than current architecture docs
- use them to understand why a recent change exists and what follow-up work was intentionally left out
- when a historical note conflicts with current docs, prefer the current docs

## Folder Structure

- one subfolder per iteration
- use a numeric prefix to preserve order
- keep each iteration focused and self-contained
